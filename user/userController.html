1. 新数据结构对应老数据结构

  • lastName：givenName
  • firstName：surname
  • sex：extension_b5c04d56d76b4ae085b6a3c7b5379c26_sex
  • DOB：Azure ad不支持，先不管
  • email：按我给你说的优先级来取
  • phoneNumber：extension_b5c04d56d76b4ae085b6a3c7b5379c26_phoneNumber
  • address1：streetAddress
  • address2：extension_b5c04d56d76b4ae085b6a3c7b5379c26_address2
  • city：city
  • postalCode：postalCode
  • provinceOrTerritory：state
  • role：extension_b5c04d56d76b4ae085b6a3c7b5379c26_role

2. 条件查询用户

按条件分页查询用户列表，条件包含：角色，名称，手机号码，email，生日

2.1. URL

GET /api/user/query/role

2.2. 参数

以下参数以表单上传：

  • lastName
  • firstName
  • phoneNumber
  • email
  • dateOfBirth
  • roleName：用户角色
    • ROLE_GRANTOR
    • ROLE_SDM
    • ROLE_LAWYER
    • ROLE_ASSESSOR
    • ROLE_THIRD_PARTY
    • ROLE_NOTARY
    • ROLE_WITNESS
  • page
  • size

2.3. 返回值

4. 选择用户角色

当用户角色为空时，需要先选择用户角色

4.1. URL

POST /api/user/choose/role

4.2. 参数

以下参数在request body提交

  • userId：user id
  • roleName：角色名称
    • ROLE_GRANTOR
    • ROLE_SDM
    • ROLE_ASSESSOR
    • ROLE_THIRD_PARTY
    • ROLE_NOTARY
    • ROLE_WITNESS

4.3. 返回值

{
    "data":null,
    "message":"success"
}
