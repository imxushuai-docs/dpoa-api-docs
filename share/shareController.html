1. 发送分享文件的验证码

分享文件前，发送验证码用于验证是否为本人操作

1.1. URL

POST /api/share/send/verifycode

1.2. 参数

无

1.3. 返回值

{
    "data":"null",
    "message":"Successful operation!"
}

2. 分享文件到email

分享poa或notary文件到指定邮箱 ，校验验证码正确才会发送

2.1. URL

POST /api/share/send2user/{verifyCode}

2.2. 参数

以下参数在请求路径上提交：

  • verifyCode：验证码

以下参数在request body中

  • entityType：分享的文件类型
    • PoA
    • Notary
  • entityId：poa id 或者 notary id
  • recipientType：接收人类型
    • SDM
    • Lawyer
    • Doctor
    • Bank
    • Third_Party
  • lastName
  • firstName
  • email
  • phoneNumber
  • faxNumber
  • organization
  • shareType：分享方式，fax或者email

2.3. 返回值

{
    "data":"null",
    "message":"Successful operation!"
}
